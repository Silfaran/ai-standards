# Command: build-plan

## Description
Executes the plan generated by the `refine-specs` command.
Reads the plan file and invokes the required agents in the defined order.

Agents may call each other during execution — for example, the Tester may request
fixes from the Backend Developer, and the Backend Reviewer may request changes
that loop back to the Backend Developer until the code meets all standards.

## Invoked by
Developer

## Agent
None — this command orchestrates the agents defined in the plan file.

## Input
The plan file generated by the `refine-specs` command:
`ai-standards/projects/{project-name}/specs/{Aggregate}/{feature-name}-plan.md`

## Steps
1. Read the plan file to understand which agents must intervene and in what order
2. Read the task file to understand the Definition of Done
3. Invoke each agent in the order defined in the plan, passing the relevant spec, task and service information
4. Allow feedback loops between agents (maximum 3 iterations per loop — if not resolved, escalate to the developer):
   - Backend Reviewer requests changes → Backend Developer fixes → Backend Reviewer re-reviews
   - Frontend Reviewer requests changes → Frontend Developer fixes → Frontend Reviewer re-reviews
   - Tester finds failing tests → corresponding Developer fixes → Tester re-runs tests
5. If DevOps changes are required, invoke the DevOps agent at the step defined in the plan
6. Verify that all Definition of Done conditions from the task file are met before completing
7. If any agent fails during execution:
   - Stop the plan immediately — do not continue with the next step
   - Report the failure to the developer with a clear description of what went wrong
   - Wait for the developer to decide whether to retry, skip, or abort
8. Report the final status to the developer

## Example flow (backend-only task)
```
Backend Developer → Backend Reviewer → (loop if changes needed) → Tester → (loop if tests fail) → Done
```

## Example flow (full-stack task)
```
Backend Developer → Frontend Developer → Backend Reviewer → Frontend Reviewer → (loops if needed) → Tester → (loops if needed) → Done
```

## Output
- All code implemented, reviewed and tested according to the plan
- Updated task file with all Definition of Done conditions marked as met
- A final status report for the developer
