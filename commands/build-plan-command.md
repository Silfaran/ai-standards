# Command: build-plan

## Description
Executes the plan generated by the `refine-specs` command.
Reads the plan file and invokes the required agents in the defined order.

## Invoked by
Developer

## Steps
1. Read the plan file
2. Read the task file (Definition of Done)
3. Invoke each agent in the defined order — each agent reads the previous agent's handoff before starting
4. Allow feedback loops (max 3 iterations per loop — if not resolved, escalate to the developer):
   - Backend Reviewer → Backend Developer → Backend Reviewer
   - Frontend Reviewer → Frontend Developer → Frontend Reviewer
   - Tester → corresponding Developer → Tester
5. After all tests pass, run `update-specs` to reconcile the spec with what was actually built
6. Delete all handoff files
7. Verify all Definition of Done conditions are met
8. Report final status to the developer

## Standard Flow (full-stack feature)
```
Tester [Phase 1: unit tests]
  → DevOps
    → Backend Developer
      → Backend Reviewer (loops if needed)
        → Frontend Developer
          → Frontend Reviewer (loops if needed)
            → Tester [Phase 2: integration tests] (loops if needed)
              → update-specs
                → Done
```

## Standard Flow (backend-only feature)
```
Tester [Phase 1: unit tests]
  → Backend Developer
    → Backend Reviewer (loops if needed)
      → Tester [Phase 2: integration tests] (loops if needed)
        → update-specs
          → Done
```

## Failure Handling
If any agent fails:
- Stop immediately — do not continue to the next step
- Report the failure to the developer with a clear description
- Wait for the developer to decide: retry, skip or abort

## Output
- All code implemented, reviewed and tested
- Updated task file with all Definition of Done conditions marked
- Updated spec file (via update-specs)
- Final status report
